Introduction

The Makerere University Fraud System is aimed at decreasing the financial shortfalls in Makerere University at large that results from a number of factors including; forgeries of tuition fees payment receipts by private students. Thus, as a team, we developed a fraud system that stores the receipt numbers for each payment made for each student to help academic registrars keep track of forged receipts within the shortest time possible.

Implementation:

We implemented the system using four programming languages i.e; PHP, JAVASCRIPT, CSS and HTML. It is assumed that Makerere has 1000 private students each having a unique student number. They are assumed to make installment payments of tuition fees in 10 months, thus 10 entries of receipt numbers. The registrar randomly selects any receipt from the 3 provided by the student, search for the receipt in the system, if found, the system returns and he simply looks at left most for the student number to confirm what is on the receipt. We randomly generated an associative array that stores student numbers and their corresponding receipt numbers. Thereafter we sorted the array using the various sort algorithms and compare their running times. We also implemented the different data structures on the generated array as elaborated in our code.

HOW OUR SYSTEM OPERATES:-

1.The interface/ Index/Home page:
First, any of our users should be able to see and interact with our homepage or what we refer to as the index page, where the user can access a navigation bar which provides links to different pages from which we implemented the system functionalities I.e Data structures, Sorting, Searching, Recursion, and the Comparison page. These elements have drop down menus that are also links to other pages. The index page also has a blog section where a user can provide feedback to us and other users about data structures topics and the fraud system at large. This is through clicking the read more button. We have exported our database called blog.sql that stores the posted information from this section Also part of the index page is a search button that takes you to the search implementation page.
2. Search Implementation: To implement the search functionality, we generated an associative array that stores both student numbers and receipt numbers. We then saved this generated array in a text file so as we don't regenerate the array every time the system runs. This is triggered by the generate button. We then generated an array from the text file and worked with it to implement the searching. By default the search button on the search.php file works using the binary search algorithm and it displays the amount of time the algorithm takes to return a given value from the array. As part of our implementation is a comparison page(comparisons.php)where we compare the performance of the binary search and linear search algorithms by subjecting the generated array to both algorithms. Their respective running time and results are also displayed. Therefore we recommend that the system uses binary search because it is more faster and more efficient especially considering the fact that Makerere university has a large number of students. Depending on the binary search algorithm, a registrar can use approximately 3.521 seconds to work on 500 students.
3.COMPARISON OF THE PERFORMANCE OF SORTING ALGORITHMS: Our system compares the performance of the following algorithms:- bubble sort, Quick sort, Merge sort, selection sort and insertion sort. These algorithms are all subjected to the generated array and their executing times shown. This is done on individual basis on every sort page I.e bubble.php, quick.php, merge.php, selection.php and insertion.php but they are compared generally on the comparisons page.
